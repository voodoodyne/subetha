SubEtha Frequently Asked Questions

What JDK version does SubEtha require?
SubEtha requires JDK1.5 or later for running and development.
What JBoss version does SubEtha require?
JBoss 4.0.4.CR2
Does SubEtha run on other J2EE appservers?

Not currently. SubEtha uses a fair amount of JBoss-specific code both in places where the J2EE spec is incomplete (or useless, ie authorization) and in places where it was very convenient (ie @Service beans and Message-Driven POJOs).

We view the appserver not as an operating system in which users run applications, but as a large library around which applications like SubEtha are built. Users install SubEtha; they should not have to care what components power it (we are ignoring political motivations of course =)).

Which databases does SubEtha support?
Any that work with JBoss EJB3 (aka Hibernate) and provide reasonable BLOB support. The works-out-of-the-box distribution will be set up to use Apache Derby by default, but detailed instructions will be provided for others. The developers are primarily using MySQL.
Does SubEtha work with my Mail Transport Agent (eg Postfix, Sendmail, Exim, Qmail)?
Yes, in fact, SubEtha currently requires an external MTA. The developers all use Postfix, but any MTA should work. See the integration page. In the future, it will be possible to use SubEtha without an MTA.
Does SubEtha support i18n (internationalization)?
SubEtha's text strings (ie: web interface, error messages, etc.) hasn't been internationalized with different languages yet (it's a future feature), but with proper configuration, it does currently support internationalized emails being forwarded through it, displayed in the archives as well as internationalized characters being set in the web interface.
